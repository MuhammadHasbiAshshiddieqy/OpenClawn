Secara default, OpenCLAWN memilih model otomatis per kompleksitas query (router cerdas — Inovasi #1). Override di bawah memaksa semua query ke satu model — berguna untuk eksperimen, mis. memakai Gemini saja.

{% if current %} Status: override aktif → {{ current[0] }} / {{ current[1] }}. Semua query memakai model ini. {% else %} Status: mode otomatis. Router memilih model per query. {% endif %} Compaction: {{ compaction }}.

Compaction (terinspirasi headroom): saat context window penuh, default memotong turn terlama (yang hilang benar-benar hilang). Mode local/cloud meringkas turn lama jadi satu blok alih-alih membuang — hemat token tanpa kehilangan konteks total. Default off karena peringkasan via LLM menambah latensi & bisa membuang nuansa; nyalakan jika percakapanmu panjang.

Catatan: model cloud (Claude/Gemini) butuh API key di .env (ANTHROPIC_API_KEY / GOOGLE_API_KEY). Model lokal (gemma4) butuh Ollama berjalan dan model sudah di-pull. Jika model terpilih tidak tersedia, OpenCLAWN turun ke fallback chain.
